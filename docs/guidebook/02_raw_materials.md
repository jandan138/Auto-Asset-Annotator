# 第二章：准备原材料

虽然 Qwen-VL 很聪明，但它有点强迫症。如果你把照片乱丢给它，它会不知所措。我们需要按规矩把原材料（3D 模型的渲染图）整理好。

## 1. 资产的“身份证”

每个 3D 模型都应该有一个独一无二的名字（ID），比如 `chair_001` 或 `monster_red_dragon`。
我们需要为每个模型建一个文件夹。

**错误的示范**：
❌ `all_images/` 里面堆满了 `chair_front.jpg`, `table_left.png`... （乱七八糟！）

**正确的示范**：
✅ 像这样整整齐齐：
```text
my_assets/              <-- 这是你的大仓库
├── chair_001/          <-- 这是第一把椅子
│   ├── front.png       <-- 正面照
│   ├── left.png        <-- 左侧照
│   ├── back.png        <-- 背面照
│   └── right.png       <-- 右侧照
├── table_999/          <-- 这是另一张桌子
│   ├── view_0.jpg
│   ├── view_1.jpg
│   └── ...
```

## 2. 视角的秘密

为了让 Qwen-VL 看清全貌，我们通常需要从 4 个角度给物品拍照：**前、左、后、右**。

但是，你的图片可能不叫 `front.png`，可能叫 `0.png` 或者 `cam_00.jpg`。
没关系！我们的工具支持**自定义别名**。

在 `config/config.yaml` 文件里，你可以告诉工具怎么找到这些图：

```yaml
data:
  views:
    # 告诉工具：“当你找‘正面’图时，如果看到 front.png 或 0.png，那都是它！”
    front: ["front.png", "0.png", "cam_00.jpg"]
    left:  ["left.png", "1.png"]
    # ...
```

## 3. 图像的小要求

*   **背景**：最好是干净的白色或灰色背景。如果背景太乱（比如在杂乱的房间里），模型可能会分心，盯着背景里的猫看。
*   **清晰度**：不要太模糊，也不用太大。512x512 像素就挺好。

## 动手时间！

现在，去创建一个测试文件夹 `test_data`，随便找几张图放进去，模拟一个资产文件夹。

```bash
mkdir -p test_data/my_first_item
# 复制几张图进去...
```

原材料准备好了，下一章我们就来点火开工！
